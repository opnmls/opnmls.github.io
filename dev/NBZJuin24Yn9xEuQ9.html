OpenMLS Configuration Guide

This guide provides a detailed overview of the configuration options available in OpenMLS. Proper configuration is crucial for optimal performance, security, and functionality of your OpenMLS instance.

Configuration File

OpenMLS uses a .env file for environment-specific configurations. This file should be located in the root directory of your OpenMLS installation. If it doesn't exist, copy the .env.example file and rename it to .env.

Configuration Options

Below is a comprehensive list of configuration options available in OpenMLS:

Option Description Default
PORT The port on which the OpenMLS server will run 3000
DB_HOST Database host address localhost
DB_PORT Database port 5432
DB_NAME Database name openmls
DB_USER Database user openmls_user
DB_PASSWORD Database password
JWT_SECRET Secret key for JWT token generation
SMTP_HOST SMTP server host for sending emails smtp.example.com
SMTP_PORT SMTP server port 587
SMTP_USER SMTP username
SMTP_PASSWORD SMTP password
ADMIN_EMAIL Email address for the system administrator admin@openmls.com
MAX_UPLOAD_SIZE Maximum file upload size in bytes 10485760 (10MB)
ENABLE_CACHING Enable or disable caching (true/false) true
CACHE_TTL Cache time-to-live in seconds 3600
LOG_LEVEL Logging level (debug, info, warn, error) info
ENABLE_RATE_LIMITING Enable or disable API rate limiting (true/false) true
RATE_LIMIT_WINDOW Rate limiting window in milliseconds 900000 (15 minutes)
RATE_LIMIT_MAX_REQUESTS Maximum number of requests per rate limit window 100

Example Configuration

Here's an example of what your .env file might look like:

PORT=3000 DB_HOST=localhost DB_PORT=5432 DB_NAME=openmls DB_USER=openmls_user DB_PASSWORD=your_secure_password JWT_SECRET=your_jwt_secret_key SMTP_HOST=smtp.gmail.com SMTP_PORT=587 SMTP_USER=your_email@gmail.com SMTP_PASSWORD=your_email_password ADMIN_EMAIL=admin@openmls.com MAX_UPLOAD_SIZE=10485760 ENABLE_CACHING=true CACHE_TTL=3600 LOG_LEVEL=info ENABLE_RATE_LIMITING=true RATE_LIMIT_WINDOW=900000 RATE_LIMIT_MAX_REQUESTS=100

Custom Fields Configuration

OpenMLS allows you to define custom fields for various entities. Custom fields are configured through the admin interface, but you can set some defaults in the .env file:

CUSTOM_FIELDS_LISTING=bedroom_count:integer,pool:boolean CUSTOM_FIELDS_AGENT=license_number:string,specialization:string

This configuration would add a 'bedroom_count' integer field and a 'pool' boolean field to listings, and 'license_number' and 'specialization' string fields to agents.

Security Considerations

When configuring OpenMLS, keep the following security considerations in mind:

  • Use strong, unique passwords for database and SMTP credentials.
  • Keep your .env file secure and never commit it to version control.
  • Regularly rotate your JWT_SECRET to enhance security.
  • Use HTTPS in production environments to encrypt data in transit.
  • Implement proper access controls and regularly audit user permissions.

Applying Configuration Changes

After making changes to your .env file, you'll need to restart the OpenMLS server for the changes to take effect. You can do this by running:

npm run restart

For more advanced configuration options and customizations, please refer to our Advanced Configuration Guide.
