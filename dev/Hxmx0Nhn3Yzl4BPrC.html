Getting Started

Welcome to OpenMLS! This guide will help you set up and start using our open-source multiple listing service platform.

OpenMLS is designed to be flexible and customizable, allowing you to create a robust real estate listing system that fits your specific needs.

Installation

To install OpenMLS, follow these steps:

  1. Clone the repository: git clone https://github.com/openmls/openmls.git
  2. Navigate to the project directory: cd openmls
  3. Install dependencies: npm install
  4. Set up the database (instructions for PostgreSQL): createdb openmls psql -d openmls -f schema.sql
  5. Configure your environment variables (copy .env.example to .env and edit as needed)
  6. Start the server: npm start

Configuration

OpenMLS can be configured through the .env file. Key configuration options include:

  • Database connection details
  • API keys for third-party services
  • Email settings for notifications
  • Custom field definitions

For a full list of configuration options, see our detailed configuration guide.

API Reference

OpenMLS provides a RESTful API for integrating with other systems. Key endpoints include:

  • GET /api/listings: Retrieve listings
  • POST /api/listings: Create a new listing
  • PUT /api/listings/:id: Update a listing
  • DELETE /api/listings/:id: Delete a listing

For full API documentation, including request/response formats, visit our API Reference.

Data Models

OpenMLS uses the following core data models:

  • Listing
  • Property
  • Agent
  • Office
  • Transaction

Each model can be extended with custom fields. See our Data Models guide for detailed information.

Custom Fields

OpenMLS allows you to define custom fields for any data model. To add a custom field:

  1. Navigate to the Admin panel
  2. Select "Custom Fields" from the menu
  3. Choose the model you want to extend
  4. Define your new field (name, type, validation rules, etc.)

Custom fields are automatically added to the database schema and API responses.

Search Functionality

OpenMLS provides powerful search capabilities, including:

  • Full-text search across all fields
  • Geospatial queries for location-based searches
  • Advanced filtering and sorting options
  • Saved searches and alerts

For details on implementing and customizing search, see our Search guide.

Integrations

OpenMLS can integrate with various third-party services and tools, including:

  • IDX/RETS feeds
  • CRM systems
  • Mapping services
  • Document management systems

Check our Integrations guide for setup instructions and best practices.

Security

Security is a top priority in OpenMLS. Key security features include:

  • Role-based access control
  • Data encryption at rest and in transit
  • Two-factor authentication
  • Audit logging

For more information on securing your OpenMLS installation, refer to our Security guide.

Troubleshooting

If you encounter issues while using OpenMLS, try these steps:

  1. Check the application logs for error messages
  2. Verify your configuration settings
  3. Ensure all dependencies are up to date
  4. Consult our Troubleshooting guide

If you still need help, don't hesitate to reach out to our community on the GitHub issues page or our community forum.
