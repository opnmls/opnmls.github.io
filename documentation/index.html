OpenMLS Documentation

Getting Started

OpenMLS is an open-source Multiple Listing Service platform designed to be flexible, scalable, and easy to integrate. This documentation will guide you through setting up, configuring, and using OpenMLS.

Installation

To install OpenMLS, you'll need to have Node.js (v14+) and MongoDB (v4.4+) installed on your system.

git clone https://github.com/openmls/openmls.git
cd openmls
npm install

Configuration

Copy the example configuration file and adjust it to your needs:

cp config.example.js config.js
nano config.js

Key configuration options include database connection details, API keys for external services, and customization options for your MLS instance.

API Reference

OpenMLS provides a RESTful API for integrating with other systems. Here are some key endpoints:

Listings

GET /api/v1/listings - Retrieve all listings

POST /api/v1/listings - Create a new listing

GET /api/v1/listings/:id - Retrieve a specific listing

PUT /api/v1/listings/:id - Update a listing

DELETE /api/v1/listings/:id - Delete a listing

Users

POST /api/v1/users/register - Register a new user

POST /api/v1/users/login - User login

GET /api/v1/users/profile - Get user profile

For a full list of API endpoints and their documentation, please refer to our API Documentation.

Data Models

OpenMLS uses MongoDB as its database. Here are the main data models:

  • Listing: Represents a property listing with fields like address, price, features, etc.
  • User: Represents a user of the system (agent, admin, etc.)
  • Agency: Represents a real estate agency
  • Transaction: Represents a property transaction

For detailed schema information, please check the Data Models Documentation.

Customization

OpenMLS is designed to be highly customizable. You can extend the platform by:

  • Adding custom fields to listings
  • Creating new API endpoints
  • Implementing custom business logic
  • Modifying the user interface

Refer to our Customization Guide for detailed instructions.

Deployment

For production deployment, we recommend using Docker. A sample docker-compose file is provided in the repository.

docker-compose up -d

For more deployment options and best practices, see our Deployment Guide.

Troubleshooting

If you encounter issues while setting up or using OpenMLS, please check our FAQ and Troubleshooting Guide. If you can't find a solution, feel free to open an issue on our GitHub repository.
